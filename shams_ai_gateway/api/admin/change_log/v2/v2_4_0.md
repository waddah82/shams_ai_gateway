### New — Skills & Agent Orchestration

- **SAG Skills subsystem** (PR #145) — new `SAG Skill` DocType lets administrators author and publish reusable LLM skills (markdown + frontmatter). Admin page gets Skills and Prompts tabs. External Frappe apps can register their own skills via the new hook API.
- **Tool response image support** (PR #123) — tool results can now include image data, so extractors and analysers can return visuals alongside text.
- **`get_pending_approvals` tool** (PR #127) — agents can discover workflow actions waiting on the current user.

### Improvements

- **Sandbox isolation for `run_python_code`** (PRs #142, #146) — user code runs in a disposable subprocess so resource limits (RLIMIT_CPU, RLIMIT_AS, SIGALRM) only affect the child, never the Frappe worker.
- **Leaner `run_python_code` sandbox** (PR #146) — matplotlib, seaborn, plotly, and scipy are no longer pre-loaded; they never rendered plots back to the caller and consumed 200-400 MB of the memory budget, causing sporadic `Memory Limit Exceeded` errors on calls like `frappe.get_doc`. `max_recursion_depth` default raised from 100 to 500 so Frappe internals don't trip it.
- **Audit log accuracy** (PR #143) — false-success entries fixed and the captured payload enriched.
- **Worker-thread timeouts** (PR #122) — signal-based timeouts are skipped when running inside a worker thread where they can't be installed.

### Security Fixes

- **Fixed** (#132) SQL injection in the removed `create_visualization` tool — tool was dead code and has been deleted.
- **Fixed** (#133) unprotected whitelist decorators on duplicate config endpoints.
- **Fixed** (#134) `extract_file_content` now enforces the parent document's permission check.
- **Fixed** (#140) usage statistics restricted to assistant admins.
