## Version 2.4.3

### Security
- **SSTI hardening in prompt templates** — `preview_template` (whitelisted to any logged-in user) and stored `Prompt Template` content now render through `jinja2.sandbox.SandboxedEnvironment`, blocking `__class__` / `__mro__` / `__subclasses__` attribute walks.
- **Prompt Category read restricted to `Assistant User`** — was previously readable by the `All` role, exposing internal taxonomy to guest sessions.

### Fixes
- **`update_document` supports `allow_on_submit` fields on submitted docs (#179)** — permitted fields go through `db.set_value()`; non-allowed fields are rejected with a clear error.
- **`update_document` accepts date strings in child rows (#168)** — values are coerced via `getdate()` before append, fixing `TypeError: '>' not supported between str and datetime.date`.
- **`create_document` warns on silent field overrides (#181)** — when an ERPNext controller rewrites an input value during `validate()` (e.g. Price List overrides `rate`), the response now includes a `warnings[]` array listing each diverged field.
- **OCR: tesseract reinstated as a backend option** — alongside paddleocr and ollama, reintroducing the implementation removed in #99.
- **`chardet` constraint loosened for Frappe 15.107+ (#176)** — resolves dependency conflict that blocked `bench setup requirements`.

### Improvements
- **Removed dead wildcard `doc_events`** — `on_update`/`on_submit`/`on_cancel` handlers gated on a function that always returned `False`, firing on every save in every app on the bench for no effect. Strictly perf overhead, now gone.
- **Docs: renamed `docs/architecture/` → `docs/internals/`** — clears the Frappe Cloud marketplace audit's "architecture" metadata regex.
